## 작업 내용

### 1. 코드 정리 (simplify.txt 기반)
- **DamagePopupPool**: SpawnDamage 파라미터 순서를 SpawnEffect에 맞춤 → `(pos, isNikke, text, color, scale)` 통일. 호출부 2곳(CombatDirector) 수정
- **NikkeData**: public 프로퍼티를 SerializeField Header 그룹 순서와 일치하도록 재정렬. CombatPrefab을 Visual 블록으로, Manufacturer/Squad/Element를 Basic Info 블록으로 이동
- **CombatFieldView → CombatDirector**: FocusIn 내 캐싱 코드 중복 → `CacheUnitView(unit, view)` 헬퍼 메서드 추출 안내

### 2. Perspective 카메라 전환 + 적 턴 기울기 효과
- 전투 씬 카메라를 **Orthographic → Perspective** (FOV 57)로 전환
- 배경만 스케일/Z거리 조절하여 기존 화면 구성 유지
- **CombatDirector**: `orthographicSize` 관련 코드 4곳을 `fieldOfView`로 변경
- **CombatCameraTilt** 새 컴포넌트 구현: 적 턴 시 카메라 Y축 -1.5도 기울기, 플레이어 턴 복귀

### 3. UI 슬롯 위치 연동
- 카메라 기울기에 따라 UI 슬롯이 어긋나는 문제 → CombatHUD에 `UpdateSlotPositionsForTilt()` / `ResetSlotPositions()` 추가
- SlotRoot(부모 RectTransform) 통째로 `WorldToScreenPoint` 기반 재배치
- ActiveTurnBar는 별도 오브젝트 → `m_CurrentTurnUnit` 캐싱 후 SlotRoot 이동 뒤 `SnapTurnBar` 재호출로 해결

## 설계 결정

- **전체 Perspective 전환 (방식 C)** 선택: 배경뿐 아니라 캐릭터도 원근 효과를 받아야 원작 DD와 유사한 연출이 나옴. 배경 전용 카메라(방식 A)나 셰이더(방식 B)는 유닛에 효과가 적용되지 않아 탈락
- **eulerAngles.y 대신 m_CurrentTiltY 직접 추적**: 음수 각도(-1.5)가 eulerAngles에서 358.5로 변환되어 한바퀴 도는 문제 방지
- **UI 갱신은 틸트 시에만 (방식 B)**: 매 프레임 WorldToScreenPoint는 과하므로, LerpTilt 코루틴 내에서만 호출하고 복귀 시 캐싱된 원래 위치로 복원
