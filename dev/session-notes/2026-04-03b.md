## 작업 내용

### 1. FocusBlur 카메라 스택 버그 디버깅
- blur 셰이더는 정상 동작하지만 포커스 유닛이 화면에 보이지 않는 문제 진단
- CombatScene.unity, TagManager.asset, FocusCameraRenderer.asset, Renderer2D.asset 분석

## 설계 결정
- **Camera Stacking 구조 검증**: Base Camera(blur, cullingMask=55) + FocusCamera Overlay(no blur, cullingMask=8) 구조는 올바름
- **FocusForeground = Layer 3**: TagManager 기준 index 3 → cullingMask bit 3 = 8, Base Camera 55(=0b00110111)에서 bit 3 제외 확인

## 발견한 버그 / 수정

| 버그 | 원인 | 수정 |
|------|------|------|
| 포커스 유닛이 렌더링 안 됨 | FocusCamera z=0 (Base Camera는 z=-10) — 스프라이트가 near clip(0.3) 이내에 위치 | FocusCamera Position z를 -10으로 변경 |
| (동반) FocusCamera Perspective 모드 | 생성 시 기본값 미변경 | Projection을 Orthographic으로 변경, Size=5.4 |

## 셰이더 학습 내용
- `SAMPLE_TEXTURE2D`: 텍스처에서 UV 좌표 1픽셀 색상 읽기
- 9-tap Gaussian: 중앙 + 직선 4방향 + 대각선 4방향 가중 평균으로 블러 구현
- 가중치 합 ≠ 1: 합 > 1이면 밝아짐(블룸 유사), 합 < 1이면 어두워짐 — 현재 코드 합 ≈ 1.07로 약간 과노출
- 블러 강도 조절: `_BlurStrength` 값 증가 또는 FullScreenPassRendererFeature 중복 추가(멀티패스)로 블록킹 없는 강한 블러 가능
