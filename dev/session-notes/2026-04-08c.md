## 작업 내용

### 1. HpBarAnimator 컴포넌트 설계 및 구현
- HP바 데미지/힐 고스트 애니메이션 연출 추가
- `HpBarAnimator` MonoBehaviour를 "Hp Bar" GameObject에 부착
- `InitHp` (애니메이션 없는 초기화) / `SetHp` (애니메이션 포함 갱신) API 분리
- 연속 피격 처리: 코루틴 진행 중 새 호출 시 `GhostFill.fillAmount` 위치에서 이어서 시작

### 2. 프리팹 3종 수정
- `EnemySlot_0`, `LargeEnemySlot_0`, `NikkeSlot_0` 각각 `Fill Area` 안에 `GhostFill` Image 추가
- GhostFill을 Fill보다 위(먼저 렌더)에 배치

### 3. CombatHUD 수정
- `Slider[]` → `HpBarAnimator[]` 교체 (Nikke/Enemy/LargeEnemy 3종)
- `RefreshHpBar` → `SetHp` 호출로 변경
- `OnBattleStarted`에서 `InitHpBar` (새 메서드) 호출로 초기화 분리

## 설계 결정

- **HpBarAnimator를 Slider와 같은 GameObject에 부착**: 책임 캡슐화, CombatHUD 변경 최소화
- **InitHp/SetHp 분리**: 전투 시작 초기화 시 불필요한 고스트 애니메이션 방지
- **데미지 고스트**: Fill 뒤 (SetAsFirstSibling) + anchorMax 줄이기 + 오렌지
- **힐 고스트**: Fill 앞 (SetAsLastSibling) + 얻은 구간(oldHp~newHp)만 알파 페이드 + 초록
  - 이유: 힐 시 메인 Fill이 즉시 커지므로 뒤에 두면 고스트가 가려짐
- **fillAmount 대신 RectTransform.anchorMax 사용**: 기존 Fill이 Simple 타입이라 Slider가 anchorMax로 제어하므로 방식 통일

## 발견한 버그 / 수정

| 버그 | 원인 | 수정 |
|------|------|------|
| InitHp NullReferenceException | `StopCoroutine` null 체크 누락 | `if (m_ActiveCoroutine != null)` 추가 후 null 처리 |
| 고스트 효과 미표시 | GhostFill Source Image가 빨간색 스프라이트 → orange tint 구별 불가 | 중립 흰색 스프라이트로 교체 |
| 고스트가 Fill 대신 메인 Fill에 적용 | `fillAmount` (Filled) vs `anchorMax` (Simple) 방식 불일치 | RectTransform.anchorMax 방식으로 통일 |
| 힐 고스트 미표시 | 힐 시 고스트가 메인 Fill 뒤에 가려짐 | isDamage 분기: 힐은 Fill 앞 배치 + 알파 페이드로 변경 |
