# 2026-03-23 세션 요약

## 작업 내용

### 1. SkillExecutor 공식 중복 제거 리팩토링
- `CalcHitChance`, `CalcCritChance`, `CalcDamageRange` 헬퍼 3개 추출
- `RollHit`, `RollCrit`, `CalcDamage`, `PreviewAttack` 모두 헬퍼 사용으로 통합
- Corpse 상태 처리(dodge=0, defense=0)를 헬퍼 안에 집중
- `CalcDamageRange`는 `(int min, int max)` 튜플 반환 → `CalcDamage`는 `Random.Range(min, max+1)`로 실제 값 롤

### 2. ApplyOnHitEffects 버그 수정
- `applied.Add(effect)`가 저항 성공/실패 무관하게 항상 호출되던 버그
- `else` 블록 안으로 이동 → 저항 실패 시에만 applied에 추가

### 3. 숫자키 스킬 선택 입력
- `SkillSelectPanel.Update()` 추가 — 1~4: 스킬 선택, 5: 이동
- `m_SkillKeys` 배열을 필드로 선언하여 매 프레임 GC 할당 방지
- 비활성 버튼(interactable=false) 키입력 무시

### 4. ESC 키 Cancel
- `TargetSelectPanel.Update()` 추가 — Escape: Cancel 버튼 동작

### 5. icon_SkillSelect 선택 표시
- 선택한 스킬 icon 위치에 indicator 표시, Cancel 시 숨김
- icon_SkillSelect를 NikkeInfoPanel/Skills 아래에 배치 (NikkeInfoPanel은 항상 표시되므로 타겟 선택 중에도 유지)
- `m_SkillIconTransforms[]` 배열로 icon_0~3 위치 참조

### 6. 숫자키로 스킬 바로 변경
- TargetSelectPanel 활성 중 1~4 키 → Cancel 없이 바로 스킬 변경
- `SkillSelectPanel.SetPendingSkill(index)` 추가 — Cancel 후 Show()에서 자동 실행
- 새로 활성화된 오브젝트의 Update()는 다음 프레임부터 실행되므로 pending 패턴으로 해결

### 7. 스킬 변경 시 preview/highlight 갱신
- 스킬 변경 후 hover 중인 적의 preview 미갱신 버그 수정
- `CombatHUD.RefreshHoveredPreview()` 추가 → `TargetSelectPanel.Show()` 마지막에 호출
- hover 상태 추적(`m_HoveredUnit`) 추가 → Show() 시 BRIGHT 하이라이트 재적용

## 설계 결정
- **pending skill 패턴**: 새로 활성화된 MonoBehaviour의 Update()가 동일 프레임에 실행되지 않는 Unity 동작 특성 때문에, SetPendingSkill + Show()에서 처리하는 방식 선택
