- ProcessSingleHit/ProcessHitBatch 공통 로직 → ProcessOneTarget 추출
- FocusIn/FocusOut lerp 루프 내 GetView() → m_ViewCache 합산 (GC 방지)
- m_FocusBuffer → HashSet<CombatUnit> 교체 (O(n) → O(1))

### 2. Focus Movement System 구현 (2026-04-01 플랜 실행)
- FocusIn: 포커스 유닛 위치/스케일 즉시 적용 (lerp 제거), 비포커스 알파 즉시 적용
- DriftRoutine: FocusIn~StopDrift 구간 동안 공격자/피격자 방향별 지속 이동
  - Melee 공격자: 전진, Ranged 공격자: 후퇴, 피격자: 항상 후퇴
- FocusOut: 스케일/위치 lerp 복귀
- SkillSequence 재배치: 상태이상/사망의문턱 팝업 → FocusOut 후로 이동
- TRIGGER_DEATH → ProcessOneTarget으로 이동 (히트 시점에 fade 시작)
- 사망 처리 블록 제거 (시체는 UnitDiedEvent → ProcessDeadUnits가 처리)

### 3. 자리이동 버그 수정
- 문제: UnitMovedEvent가 Execute() 중 발행되어 MoveAllToCurrentSlots가 Focus 도중 실행
- 수정: CombatFieldView에 SetFocusLock(bool) 추가 → Focus 중 UnitMovedEvent 무시
- FocusOut 후 SetFocusLock(false) + MoveAllToCurrentSlots() 호출
- FocusOut 위치 복귀 분기:
  - 스킬 사용자(m_FocusUser): GetSlotPosition() → 새 슬롯으로 직접 복귀
  - 이동 당한 타겟: m_OriginalPositions → 원래 위치 복귀 후 MoveAllToCurrentSlots

### 4. Camera Zoom + Overlay + Blur 시스템 구현
- 오버레이: SpriteRenderer (검정, sortingOrder=5) → 비포커스 유닛/배경 어두움 처리
- 포커스 유닛: sortingOrder → m_FocusSortingOrder(10)으로 상승, layer → FocusForeground
- Camera: orthographicSize lerp (FocusOut 시 복귀)
- 개별 유닛 알파 조작 제거 (오버레이가 일괄 처리)
- Blur: FocusBlur.shader (9-tap Gaussian) + FullScreenPassRendererFeature
  - Camera Stacking: Base Camera(blur 있음) + FocusCamera Overlay(blur 없음)
  - FocusForeground layer 유닛만 FocusCamera에서 렌더 → blur 미적용
  - FocusBlurController.cs로 BlurStrength 런타임 제어

## 설계 결정

- **오버레이 방식 선택 (Canvas X, SpriteRenderer O)**: Canvas 없이 sortingOrder만으로 제어 가능, 구조 단순
- **Blur에 Camera Stacking 선택**: FullScreenPass가 모든 것에 적용되므로 포커스 유닛을 별도 카메라로 분리하여 blur 우회
- **TRIGGER_DEATH 위치**: 히트 시점으로 이동 → Drift 중 fade 애니메이션 시작, 시체는 PlaySkillSequence 완료 후 UnitDiedEvent로 자동 처리
- **FocusIn lerp 제거**: 0.1~0.3초 lerp가 시각적으로 의미 없어 즉시 적용으로 변경
