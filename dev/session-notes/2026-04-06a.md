# 2026-04-06 세션 요약

## 작업 내용

### 1. 전투 버그 수정 - 시체에 디버프 잔존
- `SkillExecutor.Execute()`에서 `TakeDamage` 후 사망한 타겟에 `ApplyOnHitEffects`가 호출되는 문제
- `targets[i].IsAlive` 체크를 추가하여 Corpse/Dead 상태 타겟에 출혈/중독 적용 방지

### 2. Focus Drift 시스템 전면 개편
- **FocusIn 배치 문제**: 아군 스킬 시 시전자/대상이 같은 `m_NikkeFocusPoint`로 이동하여 겹치는 버그
  - 진영별 분리(`m_NikkeFocusBuffer`, `m_EnemyFocusBuffer`) 후 `AssignFocusPositions` 메서드로 배치
  - 같은 진영(아군 스킬): 화면 중앙(`screenCenter`) 기준 배치
  - 혼합 진영(적 공격): 각 진영 focusPoint 기준 배치
- **슬롯 기반 레이아웃**: 인덱스 기반 균등 분배 대신 실제 슬롯 위치의 상대 오프셋을 사용
  - `m_FocusLayoutScale`로 포커스 시 간격 조절 (Inspector)
  - 크기 2 유닛(2슬롯 점유)의 중간 위치도 자동 처리
- **적 진영 경계 보정**: `m_EnemyFocusMinXMargin`으로 적이 화면 중앙을 넘지 않도록 보정
- **Drift 방향 수정**: target의 drift 방향이 UnitType만으로 결정되어 아군-아군 케이스에서 같은 방향으로 이동하는 버그
  - 아군 단일: 시전자 뒤로, 대상 앞으로
  - 아군 전체: SlotIndex 0,1번 앞으로 / 2,3번 뒤로

### 3. Focus 시 UI 숨기기
- `CombatHUD.SetHpBarsVisible(bool)` 메서드 추가
- HP바/에블라바 부모 오브젝트(`m_NikkeHpBarRoot`, `m_EnemyHpBarRoot`)를 통째로 on/off
- `FocusIn`에서 숨기고, `FocusOut`에서 복원

### 4. Overlay 코드 제거
- blur 시스템 도입으로 overlay(`m_Overlay`, `m_OverlayAlpha`) 관련 필드/로직 제거
- `m_FocusSortingOrder`는 `AssignFocusPositions`에서 사용 중이므로 유지

### 5. 스킬 부가 이동(Move) 타이밍 변경
- `ApplyPositionMove`에서 `EventBus.Publish(UnitMovedEvent)` 제거
- 슬롯 변경은 `m_PositionSystem.Move()`로 즉시 반영하되, 시각적 이동은 FocusOut 이후 `MoveAllToCurrentSlots()`(lerp)로 처리
- FocusIn과 슬롯 이동 lerp가 동시에 발생하는 충돌 해소

## 설계 결정

- **슬롯 기반 레이아웃 채택**: 방향 벡터 계산(수직 벡터 등)으로 배치하려 했으나, 크기 2 유닛과 실제 슬롯 배열 방향이 맞지 않아 실제 슬롯 위치의 상대 오프셋 방식으로 전환
- **부가 이동 지연**: UnitMovedEvent를 제거하고 시각적 이동을 FocusOut 이후로 미뤄서, FocusIn/Drift와 슬롯 이동이 겹치지 않게 함. 플레이어 직접 Move는 별도 경로(`OnMoveTargetSelected`)로 기존 동작 유지

## 발견한 버그 / 수정

| 버그 | 원인 | 수정 |
|------|------|------|
| 시체에 출혈/중독 디버프 잔존 | `TakeDamage` 후 `IsAlive` 체크 없이 `ApplyOnHitEffects` 호출 | `IsAlive` 조건 추가 |
| 아군 힐 시 시전자/대상 겹침 | 같은 진영 유닛이 동일 focusPoint로 이동 | 화면 중앙 기준 배치 + 슬롯 오프셋 |
| 전체 공격 시 적 세로 정렬 | 수직 벡터로 spacing 계산 | 슬롯 기반 상대 오프셋으로 전환 |
| 적이 화면 중앙 넘어서 배치 | focusPoint 대비 슬롯 오프셋이 큼 | `m_EnemyFocusMinXMargin`으로 경계 보정 |
| FocusIn과 슬롯 이동 lerp 충돌 | `ApplyPositionMove`가 Execute 중 UnitMovedEvent 발행 | Event 발행 제거, FocusOut 이후 lerp |
