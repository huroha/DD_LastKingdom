# 2026-03-25 세션 요약

## 작업 내용

### 1. DeathsDoor 디버프/리커버리 로직 정리
- `hasRecovery` 죽은 코드 제거 및 후속 분기 추가
- `ApplyDeathsDoorDebuff`에 recovery 제거 로직 추가 (상호배타적 토글 구조)
- `RemoveDeathsDoorDebuff` → `ApplyDeathsDoorRecovery`로 메서드명 변경
- 공통 제거 로직을 `RemoveEffect` 헬퍼로 추출

### 2. Tooltip 시스템 GC 제거 리팩토링
- `Func<string>` → `delegate ContentBuilderHandler(StringBuilder sb)` 전환
- `TooltipTrigger`: StringBuilder 기반으로 변경, offset 파라미터 추가
- `CombatTooltip.Show`: `string` → `StringBuilder` 파라미터, `SetText(sb)` 사용 (GC-free)
- `StatusEffectIconDisplay`: `BuildEffectTooltip` void 반환으로 변경, `m_TooltipBuilder` 필드 제거
- `CombatHUD`: 모든 `SetupTooltipTrigger` 호출부를 `ContentBuilderHandler` 람다로 변경

### 3. HP/Ebla 호버 시 둘 다 표시
- HP바, Ebla바 어느 쪽을 hover해도 HP + Ebla 두 줄 모두 표시
- `sb.Append` 체인으로 `\n` 구분

### 4. Tooltip 위치/크기 개선
- 마우스 위치 기반 → HP바 RectTransform 기준 고정 위치로 변경
- `ContentSizeFitter` 제거, 코드에서 `preferredWidth/Height` 기반 직접 sizeDelta 설정
- offset을 `TooltipTrigger.Initialize` 파라미터로 전달 (HP: 위, 디버프: 아래)
- 배경 Image를 Sliced로 변경하여 늘어나도 테두리 유지

### 5. HP바 호버 영역 버그 수정
- 적 HP바 위에 TMP(unitName)가 Raycast Target 켜진 채로 가리고 있었음
- `EventSystem.RaycastAll` 디버그로 원인 특정
- TMP Extra Settings에서 Raycast Target 해제로 해결

## 설계 결정
- **`Action<T>` 대신 delegate 직접 선언** — 프로젝트 컨벤션에 따라 `ContentBuilderHandler` delegate 사용
- **ContentSizeFitter 제거** — VerticalLayoutGroup과 충돌하여 코드에서 직접 sizeDelta 제어가 더 안정적
- **DeathsDoor debuff/recovery 상호배타 토글** — 상태 전환이 보장하므로 중복 체크 불필요, Apply/Remove 양쪽에서 반대편만 제거

## 발견한 버그 / 수정
| 버그 | 원인 | 수정 |
|------|------|------|
| `hasRecovery` 세팅만 하고 미사용 | 후속 분기 누락 | `if (!hasRecovery)` 분기 추가 후 `RemoveEffect` 헬퍼로 리팩토링 |
| `ApplyDeathsDoorDebuff`에서 recovery 미제거 | recovery 제거 로직 없음 | `RemoveEffect(unit, m_DeathsDoorRecovery)` 추가 |
| 적 HP바 호버 미작동 | TMP unitName이 Raycast 가로챔 | TMP Extra Settings Raycast Target 해제 |
