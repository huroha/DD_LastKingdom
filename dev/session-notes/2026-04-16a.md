# 2026-04-16 세션 요약

## 작업 내용

### 1. Camera Shake (FocusIn 피격 시)
- `CombatFeedback`에 `PlayCameraShake()` / `ShakeRoutine()` 추가
- `Time.unscaledDeltaTime` 기반 — HitStop(`timeScale=0`) 중에도 정상 동작
- `CombatDirector`의 `ProcessSingleHit` / `ProcessHitBatch`에서 `IsHit`일 때만 호출, 회피 시 미발동
- crit 구분 없이 동일 강도

### 2. BG Tilt (FocusIn 배경 기울기)
- `CombatFocusController`에 `StartBgTilt()` / `StopBgTilt()` / `BgTiltRoutine()` 추가
- BG Transform의 Z축 회전으로 구현 — 카메라 Z-tilt 느낌이지만 FocusForeground 캐릭터에 영향 없음
- 공격 진영에 따라 방향 결정 (Nikke: +angle, Enemy: -angle)
- FocusIn 시 시작, FocusOut 시 0으로 복귀
- 기존 `CombatCameraTilt`(Y축 턴 틸트)와 독립적으로 공존

### 3. 스프라이트 교체 타이밍 변경
- 기존: FocusIn 중 attack/hit 스프라이트 설정 후 짧은 대기 뒤 즉시 idle 복귀
- 변경: FocusOut 완료 후 `RestoreSprites()`에서 일괄 복귀
- `HitSpriteRoutine`(코루틴, 자동 복귀) 삭제 → `SetHitSprite`(즉시 설정, 복귀 없음)로 대체
- `ProcessOneTarget`의 IsHit 분기 통일 — Animator Hit 트리거 분기 제거, `SetHitSprite`로 일원화
- Death 트리거는 `RestoreSprites`에서 Animator 복원 직후 발동

## 설계 결정

- **BG Transform 회전 vs 카메라 Z-tilt**: 카메라를 직접 기울이면 FocusForeground 캐릭터도 기울어지므로, BG 오브젝트 Z축 회전으로 결정. 작은 각도에서 시각적 차이 미미
- **Shake 대상을 카메라 Transform 직접 사용**: position(Shake)과 eulerAngles(CameraTilt)가 독립적이므로 별도 부모 오브젝트 불필요
- **HitSprite 방식 통일**: Animator + Hit 트리거 분기를 제거하고 `SetHitSprite`로 일원화 — FocusIn 중 idle 전환 방지를 위해. 완성본에서 모든 유닛이 Idle Animation + Hit Sprite를 갖는 구조에 맞춤
