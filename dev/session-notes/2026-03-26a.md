# 2026-03-26 세션 요약

## 작업 내용

### 1. 타겟 선택 중 마우스 클릭으로 스킬 전환 구현
- `SkillSelectPanel.ShowSkillsOnly()` 메서드 추가 — 타겟 선택 중 스킬 버튼만 노출 (Pass/Move 숨김)
- `TargetSelectPanel.TriggerCancel()` public 메서드 추가
- `OnSkillButtonClicked()`에 타겟 패널 활성 체크 분기 추가 — `SetPendingSkill` + `TriggerCancel`로 스킬 전환
- `SkillSelectPanel.Update()`에 이중 키 처리 방지 가드 추가

### 2. CombatTooltip Rich Text 색상 적용
- 니케 HP바 툴팁에 `<color=red>` (체력), `<color=white>` (에블라) TMP Rich Text 태그 적용

### 3. StatusEffectData에 Description 필드 추가
- 특정 디버프(Affliction, DeathsDoor 등) 툴팁에 설명 표시용
- `StatusEffectIconDisplay.BuildEffectTooltip()`에서 Description 출력

### 4. SkillTooltip 시스템 구현
- **SkillPositionDisplay**: UsableCircles/TargetCircles 8개 원 + ConnectionBar로 포지션 표시
  - ConnectionBar는 `LayoutElement.IgnoreLayout = true`로 HorizontalLayoutGroup에서 제외
  - 연속 범위의 첫 번째~마지막 활성 원 사이를 잇는 바
- **SkillTooltip**: StringBuilder로 전체 텍스트 조립 (스킬명, 타입, 명중, 피해보정, 크리티컬, OnHitEffects)
- **SkillSelectPanel**에 EventTrigger로 hover 이벤트 연결

## 설계 결정
- SkillTooltip 텍스트를 TMP 2개(SkillName, Description)에서 **TMP 1개 + StringBuilder** 방식으로 변경 — CombatTooltip과 동일한 구조, 크기 계산이 단순해짐
- SkillTooltipPanel 구조를 CombatTooltip과 통일: BG_Image(Stretch, IgnoreLayout) + VerticalLayoutGroup(Control Child Size Width = true)
- PositionDisplay에 `LayoutElement(Min Height 고정)`으로 VerticalLayoutGroup과의 크기 충돌 방지
- CanvasGroup(Blocks Raycasts = false)으로 툴팁 깜박임 해결 — 마우스 이벤트가 툴팁을 관통
