# 2026-03-24 세션 요약

## 작업 내용

### 1. 상태이상 아이콘 UI 표시 시스템
- HP Bar 위에 활성 상태이상 아이콘을 중앙 정렬로 표시하는 시스템 구현
- `StatusEffectIconDisplay.cs` — 오브젝트 풀링 기반 아이콘 관리 (Instantiate/Destroy 대신 SetActive로 재사용)
- 니케/적/대형적 슬롯 각각에 `HorizontalLayoutGroup` + `ContentSizeFitter` 컨테이너 추가
- `CombatHUD.RefreshHpBar()` 끝에서 `RefreshStatusIcons()` 호출하여 HP 갱신 시점과 동기화

### 2. 툴팁 시스템 (CombatTooltip + TooltipTrigger)
- `CombatTooltip.cs` — 싱글 툴팁 UI (TMP + 대화상자 Image), 마우스 위치 추적 + 화면 클램핑
- `TooltipTrigger.cs` — `IPointerEnterHandler/ExitHandler` 구현, `Func<string>` 콜백으로 동적 텍스트 생성
- 호버 대상별 콜백:
  - 니케 HP 바 → `"HP: 현재 / 최대"`
  - 니케 Ebla 바 → `"에블라: 수치 / 200"`
  - 적 HP 바 → `"남은 행동: N"` (0이면 미표시)
  - 상태이상 아이콘 → 효과명 + 도트피해/스탯변화 + 남은 차례

### 3. 상태이상 툴팁 텍스트 구성 (BuildEffectTooltip)
- `StringBuilder` 캐싱으로 GC 최소화
- 도트: `"매 차례마다 N피해 (N차례)"`
- 스탯: `"회피 -2(3차례)"`, `"피해 -25%(3차례)"` 형식
- `AppendStat` / `AppendStatPercent` 헬퍼로 일반 스탯과 % 스탯 분리

## 설계 결정
- **OnBattleStarted에서 상태이상 Clear 안 함** — 던전 복도→전투 반복 시 상태이상이 전투 간 유지되어야 하므로
- **RefreshHpBar 내부에서 RefreshStatusIcons 호출** — 모든 HP 갱신 시점에 자동으로 아이콘도 갱신되어 호출 누락 방지
- **TooltipTrigger에 Func<string> 콜백 패턴** — 각 대상마다 다른 텍스트를 유연하게 반환, 런타임 데이터 참조 가능
- **StatusEffectIconDisplay에서 풀링 + AddComponent** — 아이콘 수가 동적이므로 풀에서 재사용, TooltipTrigger는 최초 한 번만 AddComponent
