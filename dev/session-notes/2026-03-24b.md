# 2026-03-24 세션 요약

## 작업 내용

### 1. StatusEffectManager 설계 및 구현
- Pure C# 클래스, CombatStateMachine이 소유하는 구조
- `ProcessTurnStart(CombatUnit)` — DOT 틱 + DOT 지속시간 감소 + 만료 제거
- `ProcessTurnEnd(CombatUnit)` — 비DOT 효과(Buff/Debuff/Guard/Mark) 지속시간 감소 + 만료 제거
- `RemoveStun(CombatUnit)` — 스턴 제거 + Guard 해제 + 스턴저항 버프 누적 부여
- `RemoveEffectByType(CombatUnit, StatusEffectType)` — 특정 타입 효과 일괄 제거
- `DotTickResult` struct 추가 (Unit, Effect, Damage, PreviousState, ResultState)

### 2. DOT 독립 인스턴스 모델 적용
- DD 원작 참고(dev/status.txt): 출혈/중독은 같은 SO라도 별도 ActiveStatusEffect 인스턴스로 공존
- `SkillExecutor.ApplyOnHitEffects` 변경: DOT 타입이면 항상 새 인스턴스 추가 (기존 스택/갱신 로직 미적용)
- `isCrit` 파라미터 추가: 크리티컬 시 DOT 지속시간 ×1.5 올림 적용

### 3. ActiveStatusEffect 생성자 오버로드
- `ActiveStatusEffect(StatusEffectData data, int durationOverride)` 추가
- 크리티컬 DOT 지속시간 증가를 위한 duration override 지원

### 4. CombatStateMachine 통합
- RunBattle 루프 흐름: DOT 틱 → 사망 체크 → 스턴 체크 → 행동 → ProcessTurnEnd
- `ProcessDotResults` 헬퍼 추가: DOT 사망/DeathsDoor 진입 후속 처리
- 기존 CSM.RemoveStun → StatusEffectManager.RemoveStun으로 위임

### 5. StunResist_Buff SO 에셋
- Buff 타입, resistance.stun +50, Duration 1, IsStackable true
- 스턴 해제 시 매번 새 인스턴스 추가 (RecalculateStats가 스택 미반영하므로)

## 설계 결정

- **DOT 독립 인스턴스 vs 스택 합산**: DD 원작 동일 — 각 출혈/중독이 개별 지속시간으로 독립 존재. 기존 IsStackable/MaxStack은 비DOT 효과용으로 유지
- **DOT 틱 타이밍: 턴 시작 / 비DOT 지속시간 감소: 턴 종료**: Decision #28 기반. 스턴은 DOT 틱 후 별도 체크하므로 스턴 Duration이 틱에 의해 조기 소멸되지 않음
- **스턴저항 버프 누적: 새 인스턴스 추가 방식**: RecalculateStats가 CurrentStacks 미반영 → CurrentStacks++ 대신 인스턴스 추가로 StatModifier 누적
- **DeathsDoor 디버프 적용 (DOT 사망 시)**: SkillExecutor.ApplyDeathsDoorDebuff를 public으로 변경하는 대신 CSM에서 직접 ActiveEffects.Add + RecalculateStats 처리 (B안)
- **스턴 시 Guard 해제**: DD 원작 — 보호 중인 캐릭터가 기절하면 보호 해제
- **스턴저항 버프 Duration = 1**: 1라운드 후 자연 만료 (ProcessTurnEnd에서 처리)
