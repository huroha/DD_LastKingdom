# 2026-03-27b 세션 요약

## 작업 내용

### 1. Simplify Review #27~#54 진행 (simplify-review.md 후반부)

전 세션에서 진행한 Combat 시스템 최적화에 이어, UI / Data / Utils 전반의 버그·효율·컨벤션 항목을 순서대로 수정.

#### Efficiency
- **#27** `CombatHUD.RefreshTurnOrder` / `ApplyRoundText` — `ToString()` → TMP `SetText(sb)` / `SetText("{0}", int)` 오버로드로 string 할당 제거
- **#28** `CombatHUD.TickerGroup` — `[System.NonSerialized] Animator[] Animators` 추가, `CacheTickerAnimators()`로 Awake 시 캐싱. `GetComponent<Animator>()` 루프 내 호출 제거
- **#31** `EnemyInfoPanel.Populate` / `PopulatePreview` — `$""` 전면 제거, TMP `SetText` 오버로드 및 `Append(Mathf.RoundToInt(...))` 사용
- **#33** `NikkeInfoPanel` — `m_Sb` 추가, 문자열 `+` 연산 → `StringBuilder + SetText(sb)`
- **#34** `NikkeInfoPanel` — `new Color(0.4f,0.4f,0.4f,1f)` in loop → `static readonly COLOR_SKILL_DISABLED`
- **#39** `StatusEffectIconDisplay` — `m_IconPool` lazy init 제거, `Awake()`에서 초기화

#### 버그 수정
- **#1** `TurnManager.CurrentUnit` — bounds 체크 없이 인덱스 접근 → null 반환으로 안전 처리
- **#5** `CombatStateMachine.HandleEnemyTurn` — `m_CombatHUD` null 체크 누락 → null guard 추가
- **#6** `SkillExecutor.ResolveTargets` — `selectedTarget` null 체크 없이 리스트 추가 → null guard
- **#7** `CombatUnit` deathBlowResist 100% 시 사망 — `roll >= resist` → `roll > resist` (float 정밀도 + 의미 교정)

#### 컨벤션 / 캡슐화
- **#19** `SkillExecutor` 로컬 변수 `FinalResult`, `FinalDamage` → camelCase
- **#21** `CombatUnit.ActiveEffects` — `List<T>` 공개 → `IReadOnlyList<T>` + `AddEffect` / `RemoveEffectAt` / `TickCorpseTimer` 메서드
- **#22** `CombatUnit` `SlotIndex`, `TurnOrderTieBreaker`, `CorpseTimer` — public setter → private set + `SetSlotIndex` / `SetTurnOrderTieBreaker` 메서드
- **#23** `BattleStartedEvent.Nikkes` / `Enemies` — `List<T>` → `IReadOnlyList<T>`
- **#37** `TargetSelectPanel` EventBus 구독 — `Awake/OnDestroy` → `OnEnable/OnDisable`
- **#40** `RoundAnimEvent.ApplyRoundText` — null 체크 추가
- **#41** `TooltipTrigger.OnPointerExit` — `m_Tooltip` null 체크 추가
- **#43** `TargetSelectPanel` EventTrigger 셋업 3회 중복 → `AddHoverTrigger(GameObject, CombatUnitType, int)` 헬퍼 추출
- **#46** `DropTable` 필드 camelCase → PascalCase (`MinGold` 등)
- **#47** `NikkeData` 매직 넘버 `7` → `public const int MaxSkillCount = 7`
- **#54** `Singleton<T>` 필드 `m_instance` / `m_isQuitting` → `m_Instance` / `m_IsQuitting`

#### Reuse
- **#24** `IsDotEffect` — `SkillExecutor`와 `StatusEffectManager`에 중복 존재 → `StatusEffectData.cs`에 `StatusEffectTypeExtensions.IsDot()` extension method로 통합

## 설계 결정

- **EventBus `Action<T>` 예외 유지** (#53): 제네릭 유틸리티 특성상 delegate 직접 선언 컨벤션 예외 허용
- **StatBlock public 필드 유지** (#48): `[Serializable]` struct의 Inspector 편집 요구사항으로 Unity 직렬화 예외 적용
- **SetActive false→true 패턴 유지** (#30): Animator 재시작 의도적 패턴, 대안(`Animator.Play`)은 별도 작업으로 분리
- **Dict foreach 유지** (#29): C# struct 열거자 사용으로 힙 할당 없음 — false positive
- **람다 클로저 유지** (#35): `TooltipTrigger` 아키텍처 변경 필요, 스킬 실행 빈도로 영향 낮음
