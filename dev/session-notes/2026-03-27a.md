## 설계 결정

### fill-based 패턴 vs 내부 버퍼
- `PositionSystem`은 외부 호출자(CombatStateMachine, SkillExecutor, EnemyAI)가 각자 버퍼를 보유하고 넘기는 방식 채택
- `StatusEffectManager.ProcessTurnStart`는 반환 타입을 유지하면서 내부 버퍼를 반환 — 호출부(CombatStateMachine) 변경 최소화

### IsValidTarget 버퍼 분리
- `m_ValidTargetBuffer`를 `TargetSelectPanel.Show()`에 참조로 전달하는 구조상, `IsValidTarget` 내부에서 동일 버퍼를 Clear하면 패널의 `m_ValidTargets`가 비워지는 aliasing 문제 발생
- 스킬 변경 시 유효 타겟도 달라지므로 항상 최신 조회가 필요 → `m_IsValidTargetBuffer` 전용 버퍼 분리

### #13 스킵
- `SkillExecutor` `StatusEffectData[]` 타겟당 배열 — `TargetResult` 구조체 타입 변경이 필요하고 멀티 타겟 시 버퍼 오염 위험. 스킬 실행 빈도(턴당 1회)가 낮아 GC 영향 미미 → `refactoring-candidates.md`로 이동
