# 2026-04-07 세션 요약

## 작업 내용

### 1. 전투 유닛 Prefab 방식 전환
- Crown 전투 대기 애니메이션 스프라이트(01~26) 준비 후 `Crown_Idle.anim` 생성
- `Crown_CombatUnit.prefab` 생성 (SpriteRenderer + Animator)
- `NikkeData`에 `m_CombatPrefab` 필드 추가, `m_CombatAnimator` 제거
- `m_CombatIdleSprite`는 애니메이션 없는 유닛 폴백용으로 유지

### 2. CombatFieldView.CreateUnitView 개편
- 프리팹 경로 / 폴백 경로 분기 처리
  - Nikke + CombatPrefab 있음 → Instantiate 후 컴포넌트 Get
  - 나머지 → 빈 GO + SpriteRenderer (CombatIdleSprite 사용)
- Animator 유무에 따라 AnimBridge 조건부 추가
- SlotIndex 기반 sortingOrder 설정: `sr.sortingOrder = -unit.SlotIndex`

### 3. 공격/피격/회피 스프라이트 스왑 시스템
- **AttackSprite**: Animator 비활성화 → 스왑 → 재활성화 (Animator 없으면 CombatIdleSprite 복구)
- **HitSprite**: Animator 비활성화 → 스왑 → 재활성화. `previousState` 파라미터 추가로 Corpse 상태 올바르게 판단
- **회피 시 HitSprite 동일 적용**: `ProcessOneTarget`의 else 분기에 `HitSpriteRoutine` 추가
- `HasParameter` 헬퍼로 없는 트리거 호출 방지

### 4. 시체 공격 처리
- `CombatFieldView.GetView`에서 `m_CorpseViews`도 반환하도록 수정
- `FocusIn`에서 `m_FocusBuffer` 멤버 중 `m_ViewCache`에 없는 유닛(시체)을 추가로 캐싱
- `HitSpriteRoutine`에서 `previousState == Corpse`일 때 CorpseSprite 사용

### 5. EffectPopup 위치 통일
- 회피/죽음의 일격 팝업 생성 위치가 달랐던 원인: FocusIn/FocusOut 타이밍 차이
- 모든 `SpawnEffect` 호출을 `m_FieldView.GetSlotPosition(unit)` 기준으로 통일
- `SpawnEffect`에 `isNikke` 파라미터 추가, NikkeEffectOffset / EnemyEffectOffset 분리

### 6. simplify.txt 코드 정리 (10개 항목)
- **Critical**: `ProcessDotResultsRoutine`에서 `m_ActiveUnit` → `tr.Unit`
- **GC**: DriftRoutine `new Dictionary` → `m_AllyTargetDirBuffer` 멤버 캐싱
- **GC**: `SpawnDamagePopup` string 연결 → `StringBuilder` 캐싱
- **Dead Code**: `nonUserCount`, `nikkeForward` 미사용 변수 제거
- **Dead Code**: `DotTickResult.Expired` 제거 (현재 사용처 없음 확인)
- **코드 정리**: `SetFocusLock` 파라미터 오타 수정 (`locekd` → `locked`)
- **최적화**: `FocusOut` 루프를 `m_AllLivingBuffer` → `m_FocusBuffer` 직접 순회
- **코드 정리**: 상태이상/저항/죽음의 일격 이중 루프 하나로 합치기
- 7번(중복 색상 상수), 6번(Expired)은 의도적 유지로 스킵

## 설계 결정
- **SO + Prefab 병행**: SO는 데이터, Prefab은 시각 표현. `NikkeData.CombatPrefab`이 null이면 `CombatIdleSprite` 폴백
- **Animator 비활성화 방식**: Attack/Hit 스프라이트 스왑 시 `Animator.enabled = false`로 Idle 애니메이션 덮어씌움 방지
- **previousState 기반 판단**: 데미지 적용 후 unit.State가 이미 변경되므로 result.PreviousState로 시체 여부 판단
